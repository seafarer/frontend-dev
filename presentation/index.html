SASS


Watch a file
$ sass --watch input.scss:output.css

Watch a directory
$ sass --watch app/sass:public/stylesheets

Sass cont.


PROS

  • Only have to download and worry about 1 thing
  • Can use bleeding edge developments
    • data maps (json-esque feature)
    • source maps (dev tools saves  edits directly to local style sheet)
    • @at-root, &- Parent selector suffixes, Multiple assignment in @each loops
    • lots more cool stuff
    • see http://thesassway.com/news/sass-3-3-released

sass cont.


CONS

  • Have to write all your own mixins
    —thus probably reinventing the wheel
  • No vendor prefixing
  • Used alone, version conflicts between
    machines can be common

Compass

Compass is a SASS plugin that extends its functionality


Add a config.rb to your directory

$ compass init

Watch the directories you specified in your config

$ compass watch

quickly compile a change

$ compass compile

compass cont.


pros

  • Lots of mixins
  • Spriting!
  • Vendor prefixing
  • HUGE extension library
    • singularity gs
    • breakpoint
    • Sassy buttons
    • etc.

compass cont.


Cons

  • Big projects with lots of files makes 
    Compass slow as molasses
  • Poor documentation
  • Not in feature parity with latest version
    of SASS
  • Some mixins diverge from latest CSS spec

bundler

Helps keep SASS and Compass versions
in sync across machines on a per-project basis

Create a gemfile 
$ bundle init
install versions you specified in your gemfile
$ bundle install
Use bundler to compile your stylesheets 
$ bundle exec compass watch

Bundler cont


sample gemfile
source "https://rubygems.org"

group :development do
  gem "sass", "~>3.2.0"
  gem "compass", "~>0.12.0"
  gem "singularitygs", "~>1.1.2"
  gem 'rb-fsevent'
  gem 'guard'
  gem 'guard-sass'
  gem 'guard-compass'
  gem 'guard-livereload'
end

Guard


Ruby gem that compiles your stylesheets, reloads your browser and can do lots of other stuff like preprocess HAML, run JSLint, etc.


Use guard (with Bundler) to compile your stylesheets 

$ bundle exec guard

guard cont.

sample guardfile that compiles SASS 
and reloads your browser

guard 'livereload' do
    watch(%r{.+\.(sass|css|js|jpg|jpeg|png|html?|php|inc)$})
end

guard 'compass',
  :configuration_file => "config.rb" do
  watch(%r{scss/.+\.s[ac]ss})
end

Zurb Foundation

Foundation comes chock full of goodies.

Pros

  • Tons of out-of-the-box responsive styles / scripts / styling
  • Great for big sites

Cons

  • Small sites might use 10% of the code
  • Dont want to fight the framework
  • Tends to produce sites that look the same

LIBSASS

A C /C++ version of SASS.
Written by SASS original author.

Primarily compiled with Node.js
but many other implementations are available

libsass Cont.


Pros

  • 10x faster to compile than Ruby Sass
  • 100x faster to compile than Ruby Sass
  • 1000x faster to compile than Ruby Sass
  • Funded and actively in development: 
    seeking feature parity with Ruby Sass

libsass cont.


cons

  • Not compatible with Compass
  • Current feature parity ~ SASS 3.2
  • Requires Node.js

Bourbon

Bourbon is a lightweight mixin library
bourbon.io

Install the gem 
$ gem install bourbon 
Install Bourbon into your project's stylesheets directory  
$ bourbon install

add to your stylesheet 

@import 'bourbon/bourbon';

bourbon cont.

Compared to Compass:

  • Bourbon doesn't rely on Ruby;
    Its just mixins
  • Compass is a Ruby extension framework
    that extends Sass functionality
  • Both use same or similar syntax
  • Bourbon doesnt have:
    • spriting
    • ???

GRunt

A Nodejs/NPM-based task runner and
dependency manager. Similar to Guard/Bundler.

instead of a gemfile and a guardfile
you use package.json & Gruntfile.js

install dependencies 
$ npm install
start task runner 
$ grunt

sample package.json

 {
  "name": "bmi",
    "version": "0.0.1",
    "devDependencies": {
        "grunt": "0.4.1",
        "grunt-contrib-watch": "0.4.3",
        "grunt-sass": "0.12.1"
    }
}

Sample Gruntfile.js

 module.exports = function(grunt) {
    grunt.initConfig({
        pkg: grunt.file.readJSON('package.json'),
        watch: {
            sass: {
                files: ['sass/**/*.{scss,sass}' ],
                tasks: ['sass:dist']
            },
            livereload: {
                files: ['*.html', '*.php', 'js/**/*.{js,json}', 'css/*.css', 'images/**/*.{png,jpg,jpeg,gif,webp,svg}'],
                options: {
                    livereload: true
                }
            }
        },
        sass: {
            dist: {
                options: {
                    outputStyle: 'expanded'
                },
                files: {
                    'css/bmi.css': 'sass/bmi.scss'
                }
            }
        }
    });
    grunt.registerTask('default', ['sass:dist', 'watch']);
    grunt.loadNpmTasks('grunt-sass');
    grunt.loadNpmTasks('grunt-contrib-watch');
};

Autoprefixer

Forget mixin libraries. Just use autoprefixer.


a {
    transition: transform 1s
}

becomes

a {
    -webkit-transition: -webkit-transform 1s;
    transition: -ms-transform 1s;
    transition: transform 1s
}

Install autoprefixer

For your project

$ npm install grunt-autoprefixer --save-dev


globally

$ npm install grunt-autoprefixer -g

Sample Gruntfile.js

module.exports = function (grunt) {
    'use strict';

    grunt.initConfig({
        autoprefixer: {
            options: {
                browsers: ['last 2 versions']
            },
            dist: {
                files: {
                    'style.css': 'style.css'
                }
            }
        },

        watch: {
            styles: {
                files: ['style.css'],
                tasks: ['autoprefixer']
            }
        }
    });

    grunt.loadNpmTasks('grunt-autoprefixer');
    grunt.loadNpmTasks('grunt-contrib-watch');
};
